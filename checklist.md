# ✅ SEO Checklist para Landing Page (Astro)

## 🟢 1. Esenciales

* [ ] Definir `<title>` con keyword principal
* [ ] Agregar `<meta name="description">`
* [ ] Usar un solo `<h1>`
* [ ] Explicar claramente qué ofreces
* [ ] Usar dominio propio

---

## 🟢 2. Estructura del contenido

* [ ] Usar `<h1>` para el tema principal
* [ ] Usar `<h2>` para secciones
* [ ] Usar `<h3>` para detalles
* [ ] Incluir keywords de forma natural

---

## 🟢 3. Performance

* [ ] Optimizar imágenes (WebP/AVIF)
* [ ] Usar lazy loading en imágenes
* [ ] Minimizar JavaScript innecesario
* [ ] Mantener tiempo de carga < 3s
* [ ] Medir con PageSpeed Insights

---

## 🟢 4. SEO técnico

* [ ] Generar `/sitemap.xml`
* [ ] Crear `/robots.txt`
* [ ] Verificar HTTPS activo
* [ ] Usar URLs limpias (`/contacto`)

---

## 🟢 5. Open Graph (redes sociales)

* [ ] Agregar `og:title`
* [ ] Agregar `og:description`
* [ ] Agregar `og:image`

---

## 🟢 6. Indexación

* [ ] Registrar sitio en Google Search Console
* [ ] Subir sitemap
* [ ] Solicitar indexación manual
* [ ] Revisar errores de rastreo

---

## 🟢 7. Contenido

* [ ] Explicar claramente el producto/servicio
* [ ] Responder dudas del usuario
* [ ] Usar keywords relevantes
* [ ] Evitar texto genérico

---

## 🟢 8. Conversión

* [ ] CTA claro (botón principal)
* [ ] Formulario o contacto visible
* [ ] Mensaje directo de valor

---

## 🟢 9. Analytics

* [ ] Instalar Google Analytics (u otra herramienta)
* [ ] Medir tráfico
* [ ] Medir conversiones

---

## 🧠 Tips PRO

* [ ] Publicar rápido y mejorar iterando
* [ ] Probar diferentes títulos
* [ ] Conseguir backlinks básicos
* [ ] Medir y ajustar constantemente

---

## ⚡ Flujo de trabajo

1. Hacer cambios en Astro
2. Hacer push a GitHub
3. Deploy automático (Netlify/Vercel)
4. Revisar performance
5. Solicitar indexación en Search Console
6. Repetir
